# Requerimientos Funcionales - MazBasura

## Introducción

Este documento presenta los requerimientos funcionales principales del prototipo MazBasura, una aplicación móvil académica orientada a mejorar la consulta de información sobre el servicio de recolección de basura en Mazatlán.

## Requerimientos funcionales

### RF01 - Consultar rutas de recolección

El sistema debe permitir al usuario visualizar las rutas de recolección disponibles dentro de la ciudad.

### RF02 - Consultar horarios

El sistema debe mostrar los horarios de recolección correspondientes a cada zona o colonia.

### RF03 - Visualizar ubicación del camión

El sistema debe permitir consultar la ubicación aproximada del camión recolector mediante un mapa.

### RF04 - Registrar direcciones favoritas

El sistema debe permitir al usuario guardar una o más direcciones favoritas para consultar información relacionada con su zona.

### RF05 - Recibir alertas

El sistema debe enviar notificaciones o alertas al usuario antes de que el camión recolector pase por su zona.

### RF06 - Consultar información del servicio

El sistema debe mostrar información general sobre el servicio de recolección, como horarios, recomendaciones y avisos importantes.

### RF07 - Reportar incidencias

El sistema debe permitir que el usuario reporte problemas relacionados con la recolección de basura, como retrasos, basura acumulada o falta de servicio.

### RF08 - Gestionar perfil de usuario

El sistema debe permitir al usuario consultar y modificar información básica de su perfil.

## Requerimientos no funcionales

### RNF01 - Facilidad de uso

La aplicación debe contar con una interfaz sencilla, clara e intuitiva para que cualquier usuario pueda utilizarla.

### RNF02 - Disponibilidad

La información del servicio debe estar disponible para los usuarios cuando la necesiten.

### RNF03 - Diseño responsivo

El prototipo debe estar pensado para dispositivos móviles.

### RNF04 - Seguridad básica

La información del usuario debe manejarse de forma segura y no debe exponerse públicamente.

## Conclusión

Los requerimientos definidos permiten establecer las funciones principales del prototipo MazBasura y sirven como base para el diseño de la aplicación móvil.
